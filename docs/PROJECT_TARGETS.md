# SysQ 项目目标定义

## 1. 业务目标

1. 以统一框架完成从数据处理、模型训练、策略回测到每日交易计划生成的全链路闭环。
2. 通过影子账户与实盘账户并行跟踪，持续评估策略在真实执行环境中的偏差。
3. 支持小资金与常规资金两类场景，确保策略在不同账户规模下可执行。

## 2. 工程目标

1. 核心模块接口稳定，关键 API 受契约文档约束。
2. 每次改动可验证，至少通过基础编译检查与自动化测试。
3. 文档可维护，项目目标、环境定义、开发流程独立于特定 IDE 工具。

## 3. 交付目标

1. 标准化产物：
   - 数据产物：`data/qlib_bin`
   - 模型产物：`data/models/*`
   - 交易计划：`data/plan_{date}_{account}.csv`
2. 标准化脚本入口：
   - 训练：`scripts/run_train.py`
   - 回测：`scripts/run_backtest.py`
   - 每日交易：`scripts/run_daily_trading.py`
3. 标准化测试入口：
   - 全量：`python -m unittest discover tests`

## 4. 非目标范围

1. 不在当前阶段直接对接真实下单 API 自动执行。
2. 不在当前阶段覆盖高频交易与毫秒级撮合仿真。
3. 不在当前阶段承诺跨市场通用策略模板。
