# ADR-002: Use Tushare Free First

## 状态

已采纳

## 背景

项目早期目标是先跑通完整链路并建立可验证工程基线，而不是先追求高成本数据能力。直接接入付费数据源会带来：

- 初始化成本和权限依赖增加。
- 协作成员难以复现实验。
- 调试周期变长，问题定位变复杂。

## 决策

优先支持 Tushare 免费数据能力，先确保核心链路稳定：

- 数据接入可运行。
- 特征与模型训练可运行。
- 回测和每日计划可运行。

在此基础上，再逐步扩展到更高质量或付费数据源。

## 影响

正向影响：

- 上手门槛低，便于快速复现。
- 工程稳定性优先，减少早期外部依赖扰动。

代价：

- 数据覆盖和质量受限，部分策略表现受约束。

## 后续

- 当主链路稳定后，评估并引入可切换的数据源抽象层。
