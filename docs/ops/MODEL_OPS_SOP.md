# 模型操作 SOP

## 1. 目的

让模型维护可重复、可回滚，不把研究结果直接当生产结果上线。

## 2. 输入

- 生产 manifest
- 候选模型目录
- 训练窗口
- 严格评估结果

## 3. 输出

- 当前生产模型路径
- 候选与生产对比结论
- 是否准许切换
- 需要的回滚点

## 4. 标准步骤

### Step A：确认当前生产模型

通过条件：

- manifest 可解析
- 模型目录存在
- 元信息可读

### Step B：决定是否需要重训

建议触发条件：

- 模型过期
- 数据口径显著变化
- 周级维护窗口到期

### Step C：执行训练与严格评估

至少保留：

- 训练窗口
- 特征集
- 严格评估窗口
- 收益 / 回撤 / Sharpe / 交易次数

### Step D：人工审批后再切 production

规则：

- 不允许“因为是最新目录”就自动上生产
- 必须能说清：为什么替换、比谁好、坏了怎么回滚

## 5. 成功标准

- 生产模型可解释
- 候选模型有评估依据
- 替换前后都有可追踪记录

## 6. 常见故障

### manifest 指向不存在目录

处理：修 manifest 或回滚到上一个可用模型。

### 模型元信息缺失

处理：补齐训练摘要与模型 metadata，再允许进入盘前。

### 研究结果污染生产

处理：研究目录与生产 manifest 分离，未经审批不切换。

## 7. 人工接管

人工接管模型链路时，至少写明：

- 当前 production 指向
- 候选模型路径
- 评估摘要
- 是否回滚
- 回滚后需验证的盘前样例
